# 画版Pro - 技术架构文档

## 1. 项目概述

画版Pro是一个功能强大的像素艺术编辑器，专为Windows系统设计，使用EasyX图形库实现图形界面。项目采用模块化、面向对象的设计理念，具有良好的可扩展性和可维护性。

## 2. 技术栈

| 技术/库 | 版本 | 用途 | 来源 |
|---------|------|------|------|
| C++ | C++11 | 主要开发语言 | 标准库 |
| EasyX | 20240913(beta) | 图形界面库 | [EasyX官网](https://easyx.cn/) |
| CMake | 3.10+ | 构建系统 | [CMake官网](https://cmake.org/) |
| Win32 API | Windows | 系统调用 | Windows SDK |
| STL | C++11 | 数据结构管理 | 标准库 |

### 3.2 核心模块说明

#### 3.2.1 Canvas 模块
- **职责**：管理像素数据、图层系统、画布变换（缩放/平移）
- **核心类**：`Canvas`、`LayerManager`、`Layer`
- **关键功能**：
  - 32x32 像素画布管理
  - 多图层支持（创建、删除、复制、重排序）
  - 图层透明度控制
  - 画布缩放（100%-400%）
  - 画布平移（Ctrl+拖动）
  - 网格显示控制

#### 3.2.2 Tools 模块
- **职责**：实现各种绘图工具
- **核心类**：`Tools`
- **支持的工具**：
  - 画笔（Pen）
  - 橡皮擦（Eraser）
  - 油漆桶（Bucket）
  - 直线（Line）
  - 矩形（Rectangle）
  - 圆形（Circle）
- **工具参数**：
  - 画笔大小（1x/2x/3x）
  - 颜色选择

#### 3.2.3 History 模块
- **职责**：实现撤销/重做功能
- **核心类**：`History`
- **关键功能**：
  - 最多50步历史记录
  - 完整的图层状态保存和恢复
  - 支持所有编辑操作的撤销/重做

#### 3.2.4 UI 模块
- **职责**：实现用户界面
- **核心类**：`UI`、`Button`
- **界面元素**：
  - 工具栏（工具选择、画笔大小）
  - 调色板（16色预设 + 最近使用颜色）
  - 图层列表（显示、切换、管理图层）
  - 功能按钮（保存、导入、撤销/重做等）
  - 信息显示（快捷键提示）

#### 3.2.5 Animation 模块
- **职责**：实现帧动画功能
- **核心类**：`Animation`、`AnimationFrame`
- **关键功能**：
  - 帧的添加、删除、复制
  - 帧的预览和切换
  - 动画播放控制
  - GIF导出（基础实现）

#### 3.2.6 FileIO 模块
- **职责**：处理文件导入/导出
- **核心类**：`FileIO`（静态类）
- **支持的格式**：
  - 图像导入/导出（PNG/BMP）
  - 动画导出（GIF）

#### 3.2.7 Filter 模块
- **职责**：实现图像处理滤镜
- **核心类**：`Filter`（静态类）
- **支持的滤镜**：
  - 亮度调整
  - 对比度调整
  - 模糊效果
  - 锐化效果
  - 灰度转换
  - 颜色反相

#### 3.2.8 Plugin 系统
- **职责**：支持第三方扩展
- **核心类**：`PluginManager`、`Plugin`（接口）
- **支持的扩展类型**：
  - 工具扩展
  - 滤镜扩展
  - 菜单扩展

## 4. 代码质量与设计原则

### 4.1 设计原则

1. **单一职责原则**：每个模块只负责一个特定功能
2. **面向对象**：使用类封装功能，消除全局变量依赖
3. **模块化设计**：将功能拆分为独立模块，便于维护和扩展
4. **低耦合**：模块间通过明确的接口通信，减少依赖
5. **高内聚**：相关功能集中在同一模块中

### 4.2 代码质量特性

- **可读性**：清晰的命名规范、适当的注释
- **可维护性**：模块化设计、代码组织合理
- **可扩展性**：插件系统支持功能扩展
- **可靠性**：完善的错误处理、历史记录系统
- **性能**：高效的像素操作算法、批量绘制

### 4.3 关键设计决策

1. **图层系统**：采用分层设计，支持复杂的图像编辑
2. **历史记录**：使用状态保存机制，支持完整的撤销/重做
3. **工具系统**：统一的工具接口，便于添加新工具
4. **插件架构**：基于接口的插件系统，支持第三方扩展
5. **构建系统**：使用CMake实现跨编译器支持

## 5. 性能优化

### 5.1 渲染优化
- **批量绘制**：使用EasyX的批量绘制功能减少重绘次数
- **区域更新**：只重绘变化的区域
- **缩放算法**：优化像素缩放算法，保证图像质量

### 5.2 内存管理
- **对象池**：重用临时对象，减少内存分配
- **图层数据**：按需加载图层数据，节省内存
- **历史记录**：合理限制历史记录数量，避免内存占用过大

### 5.3 算法优化
- **填充算法**：使用优化的洪水填充算法
- **几何绘制**：使用Bresenham算法绘制直线
- **滤镜处理**：使用高效的图像处理算法

## 6. 扩展性设计

## 9. 技术亮点

1. **模块化架构**：从1600+行的单文件拆分为多个独立模块，提高可维护性
2. **图层系统**：支持多图层编辑，包括创建、删除、复制、重排序等功能
3. **历史记录**：完整的撤销/重做系统，支持所有编辑操作
4. **动画系统**：支持帧动画编辑和GIF导出
5. **插件架构**：基于接口的插件系统，支持第三方扩展
6. **优化的构建系统**：支持Visual Studio、CMake、批处理和Python脚本多种构建方式
7. **高性能渲染**：使用批量绘制和区域更新优化渲染性能
8. **跨平台潜力**：模块化设计便于迁移到其他平台
